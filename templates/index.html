Proyecto orientado a optimizar el flujo vehicular mediante análisis de datos y semáforos inteligentes.

1. Introducción

Concepto

El proyecto se enmarca en el desarrollo de una Ciudad Inteligente, buscando optimizar la gestión urbana mediante herramientas tecnológicas. En esta primera fase se aborda el subsistema de semaforización inteligente, orientado a mejorar el flujo vehicular con análisis de datos en tiempo real.

Tecnología

Se diseñó una base de datos relacional que almacena información de sensores, vehículos, ubicaciones y fechas. Los datos fueron procesados mediante ETL (Extract, Transform, Load) con SQL Server Integration Services (SSIS), garantizando integridad y consistencia.

Resultados

Se construyó una base multidimensional para análisis y evaluación del tráfico, sentando las bases para la toma de decisiones. Se realizaron pruebas de rendimiento comparando modelos relacional y multidimensional, evaluando su eficiencia y capacidad analítica.

Contexto

El proyecto se centra en la movilidad de la ciudad de Bogotá, una de las más congestionadas del mundo. El sistema toma datos cada 5 minutos, las 24 horas del día, sobre el flujo vehicular y la velocidad en cada semáforo. Mediante procedimientos almacenados, el tiempo de los semáforos se ajusta dinámicamente para reducir la congestión y mejorar la movilidad urbana.

2. Objetivos

Objetivo General

Desarrollar la primera fase del sistema de información para la Ciudad Inteligente, enfocada en la creación, carga y modelado de una base de datos relacional y multidimensional que soporte el análisis del tráfico vehicular mediante semáforos inteligentes.

Objetivos Específicos
  • Diseñar un modelo relacional que represente las entidades principales del sistema: vehículos, ubicaciones, fechas, semáforos y velocidades.
  • Implementar procesos ETL en SSIS para la carga controlada de los datos hacia la base de datos SQL Server.
  • Crear un modelo multidimensional derivado del modelo relacional, optimizado para análisis y consultas de alto nivel.
  • Realizar consultas comparativas entre ambos modelos y medir los tiempos de respuesta para evaluar el rendimiento.
  • Identificar y documentar errores o dificultades surgidas durante la carga y la ejecución de las consultas, proponiendo posibles soluciones.

3. Modelos Relacionales

Título ejemplo - Esquema y Normalización

El modelo relacional está normalizado (3FN) para asegurar integridad y eficiencia en la carga/consulta. Se organiza alrededor de entidades operacionales (lecturas de tráfico y estados de semáforo) y catálogos de referencia (tiempo, ubicación, dispositivos y tipos).

  • Integridad: claves primarias (PK), claves foráneas (FK) y dominios controlados.
  • Historial: timestamps, origen de datos y bitácoras de carga (ETL).
  • Consultas: vistas para validación, agregación y control de calidad.
